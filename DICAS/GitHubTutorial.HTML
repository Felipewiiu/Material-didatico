Tutorial para aprendizado e manipulação do GitHub

Para iniciar um novo repositório é usado o comando (git init) --> vai ser criado um arquivo oculto na pasta onde o git foi iniciado. para saber se o repositório que foi criado (o arquivo oculto) está atualizado com os arquivos use o comando (git status) COMMITS == são gravações das alterações dos estados dos arquivos no repositório === CRIAÇÃO DE UMA NOVA VERSÃO BRANCHS == São a mesma coisa que criar uma cópia do repositório na linha do tempo da arvore do projeto no GitHub Na parte de adição de arquivo no repositório via comando, para adicionar vários ao mesmo tempo basta usar o comando (git add .)--> é o preparo para o COMMIT

Para enviar os arquivos para o repositório remoto se usa o comando (git push) --> faz o upload para o GitHub COMMIT -M == Esse parâmetro (-M) é usado para colocar uma mensagem como explicação do commit ex: git --- ( comit -m "commit inicial" ) GIT REFLOG == serve para identificar as sequência de histórico de modificação do código --> a versão mais atual vem no topo da descrição Partindo do ponto que se cometeu um erro no código, é possível retornar a uma versão anterior graças aos commits que foram realizados. O comando para se retorna a versão anterios é (git reset --hard) usando a id que é disponibilizada no comando (GIT REFLOG) EX: 8dda315 --> ID

Processo de criação de BRANCH

Uma branch é o local onde se é guardado o código de vercionamento ==> como cópias do arquivos -- é possivel ter varia BRANCHS no caso esse procedimento é viável se o código é compartilhado com mais de uma pessoa que esteje trabalhando no seu desenvolvimento Para verificar quantas branchs existem use o comando (git branch) Para a criação de uma nova branch via comando basta digitar (git branch ++ o nome que deseja dar para ela) Para saber qual branch está trabalhando ela estará marcada com * e com a cor (verde) para fazer a alteração por tipo de branch use o comando (git checkout ++ nome da branch)--> isso irá trocar a branch que está trabalhando

UNIÃO DE BRANCHS COM O REPOSITÓRIO FINAL

ANTES DE FAZER QUALQUER COISA É PRECISO TRAZER O ARQUIVO MAIS ATUALIZADO DO GITHUB PARA O REPOSITÓRIO LOCAL COM O COMANDO (git pull) somente depois dessa ação se deve fazer o comando (merge) Para fazer a união use o código (git merge + o nome da branch) na branch principal onde deseja puxar as atualizações e fazer a união

PASSOS PARA INICIAR O REPOSITÓTIO NA GITHUB

  1. GIT pull da branch principal
  2. Gerar uma nova branch a partir da branch principal
  3. Trabalhar e adicionar novas funcionalidades na nova branch que criou
  4. Finalizar o trabalho na branch temporária
  5. Git checkout na branch principal
  6. Git pull
  7. Mergia (unir) o código da branch temporário com a branch principal (depois de testar)
  8. Fazer o git push da branch principal
